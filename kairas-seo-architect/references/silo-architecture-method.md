# Metodología de Arquitectura en Silos para KAIRAS

## Qué es un silo de contenidos

Un silo es un conjunto de contenidos agrupados por tema que se enlazan entre sí, formando una unidad temática coherente. Google entiende que un sitio que tiene múltiples contenidos relacionados sobre un mismo tema tiene autoridad real sobre ese tema.

**Estructura de un silo:**
```
Página pilar (la más completa y autorizada del silo)
  ↑ recibe enlaces de todos los posts satélite
  ↓ enlaza a páginas de servicio o contacto
  
Posts satélite (desarrollan subtemas del silo)
  → cada uno enlaza a la pilar
  → cada uno puede enlazar a otros satélites relacionados
  → todos enlazan a la página de servicio o CTA final
```

## Reglas de enlazado interno por silo

### Jerarquía de autoridad

```
Página de inicio (máxima autoridad)
  → Páginas de servicio (segunda capa — reciben links desde home y posts)
    → Páginas pilar de silo (tercera capa — reciben links de posts satélite)
      → Posts satélite (cuarta capa — enlazan hacia arriba siempre)
```

### Reglas de anchor text

| Regla | Ejemplo correcto | Ejemplo incorrecto |
|-------|-----------------|-------------------|
| Descriptivo y natural | "automatización de citas para clínicas" | "haz clic aquí" |
| Variado entre páginas | Usar 3–4 anchors distintos para la misma URL | Siempre el mismo anchor exacto |
| Sin sobreoptimización | Anchor semántico: "gestión automática de pacientes" | Keyword exacta repetida 5 veces |
| Evitar anchor genérico | "gestión de recordatorios" | "ver más", "este artículo", "aquí" |

### Cantidad mínima de enlaces internos

| Tipo de página | Mínimo de enlaces internos recibidos | Mínimo de enlaces que da |
|----------------|--------------------------------------|--------------------------|
| Página de inicio | — | 5–10 (a servicios y pilares) |
| Página de servicio | 3+ | 3–5 (a posts relacionados, contacto) |
| Página pilar | 5+ (de posts satélite) | 2–4 (a servicio, contacto, otros silos) |
| Post satélite | 1+ | 2–4 (a pilar, a servicio, a otro satélite) |

## Prevención de canibalización

### ¿Cuándo hay canibalización?

Cuando dos páginas de kairas.es tienen:
- La misma keyword principal o muy similar
- La misma intención de búsqueda
- Contenido solapado sin diferenciación clara

### Cómo prevenirla al diseñar el silo

1. **Una keyword = una página.** Si dos posts quieren posicionar "automatización clínicas", uno debe tener una variación diferente: "automatización de citas clínicas" vs "automatización de WhatsApp clínicas".

2. **Intención diferente = intención diferente.** "Qué es la automatización para clínicas" (informacional) y "contratar automatización para clínicas en Galicia" (transaccional) no se canibalizan aunque suenen similares.

3. **Revisar antes de crear.** Antes de proponer un nuevo contenido, buscar en kairas.es si ya existe algo similar. Si existe, proponer mejorarlo o ampliarlo, no crear uno nuevo.

4. **El mapa de keywords es la referencia.** Mantener actualizado un mapa donde cada URL tenga asignada su keyword principal. Si una keyword ya está asignada, no asignarla a otra página.
